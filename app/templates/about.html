Escalonamento

Um ponto crucial no desenvolvimento de um Sistema Operacional Multiprogramável é como alocar de forma eficiente o processador (CPU) para os vários processos prontos para serem executados;

Para implementar o compartilhamento da CPU entre os diversos processos no estado pronto, deve existir um critério para determinar, qual o próximo processo a executar;

Objetivos de um escalonamento

  • Maximizar uso da CPU;
  • Maximizar a produtividade;
  • “Justiça” entre os processos;
  • Minimizar tempo de resposta.

Escalonamento – Não Preemptivo

Quando o processador alocado para um determinado processo não pode ser retirado deste até que o processo seja finalizado ou seja bloqueado.

Algoritmo FIFO

O primeiro processo que solicita a CPU é o primeiro a ser selecionado para execução;

Os processos que estão prontos para serem executados pela CPU são organizados numa fila, que funciona baseado na política FIFO (First in First out – Primeiro a entrar é o primeiro a sair).

Algoritmo Menor Tarefa Primeiro

O processo que tem o menor ciclo de processamento (tempo de execução) é selecionado para usar o processador.

O processo executando não perde o processador para um processo de menor duração.

Escalonamento – Preemptivo

Quando o processador alocado para um determinado processo pode ser retirado deste em favor de um outro processo.

Algoritmo Round Robin

Conhecido também como algoritmo de escalonamento circular, também organiza a lista de processos prontos como uma fila simples, semelhante ao algoritmo FIFO;

O primeiro processo na fila de prontos recebe o processador;

No entanto, cada processo recebe uma fatia de tempo do processador, comumente chamado de quantum.

Um processo executa durante um quantum específico. Se o quantum for suficiente para este processo finalizar, outro processo do início da fila é selecionado para executar. Se durante sua execução o processo se bloquear (antes do término do quantum), outro processo da fila de prontos também é selecionado. Se terminar a fatia de tempo do processo em execução, a CPU sofre preempção e fica disponível para outro processo.

Algoritmo por Prioridades

os processos são organizados na fila de prontos baseado em prioridades.

A cada processo é atribuído uma prioridade;

O processo com maior prioridade na fila de prontos receberá o processador;

O processo que está executando pode ser interrompido pela chegada na fila de prontos de um processo de maior prioridade.

Algoritmo de Múltiplas Filas

Prioridades são atribuídas à classe de processos (de acordo com o tipo de processamento), cada qual com sua própria fila de prontos com seu mecanismo de escalonamento;

Processos das classes de maior prioridade recebem o processador;

Os processos das classes de menor prioridade só receberão o processador se as filas de prontos das outras classes (de maior prioridade) estiverem vazias;

Pode ser realizado retroalimentação em multiníveis.
